# Projects

Product sees "what customers want"; engineering sees "what machines can do". This gap compounds from team miscommunication into market-wide coordination failures. (see: [On Tech Debt and Market Failure](/field-notes/on-tech-debt-and-market-failure))

The solution: bridge business intent with engineering execution.


### [Mercator ↗](https://mercator.bicameral-ai.com)

Automated RFQ for software contracting. Chatbot that produces instant quotes for buyers and clear tech specs for contractors.

### [Pico ↗](https://pico.bicameral-ai.com)

Code reviewer to detect feature requirement gaps. Integrated with meeting transcript software and GitHub PRs.

---

### [Cosimo ↗](https://cosimo.bicameral-ai.com)

Goal alignment tool that captures _why_ your tasks matter to help with prioritization. Integrated with Claude MCP to link objectives to deliverables with explicit reasoning.
